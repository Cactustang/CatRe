# CatRe 猫咪行为记录 App 方案

## 1. 项目定位

CatRe 是一款面向养猫用户的猫咪护理与行为打卡 App。核心目标是帮助用户记录洗澡、驱虫、梳毛、剪指甲、喂药、称重、就医等行为，并通过首页统计、频率提醒和日历视图，让用户快速知道“上次做了什么”“多久没做了”“近期做了几次”。

第一版建议优先做 Android 本地版，保证离线可用、操作简单、统计清晰。后续再扩展云同步、多成员共享、图片记录和智能提醒。

## 2. 技术路线

### 2.1 推荐技术栈

- 开发语言：Kotlin
- UI 框架：Jetpack Compose
- 架构模式：MVVM
- 本地数据库：Room + SQLite
- 异步数据流：Kotlin Flow + Coroutines
- 依赖注入：Hilt，可在第二阶段引入
- 定时提醒：WorkManager + Android Notification
- 图片加载：Coil
- 图表统计：Vico 或 MPAndroidChart，MVP 可先用数字卡片替代
- 构建工具：Gradle + Android Gradle Plugin

### 2.2 为什么选择 Kotlin + Compose

- 适合快速开发 Android 原生应用。
- Compose 做打卡按钮、统计卡片、日历视图更灵活。
- Room 对本地记录类 App 很稳定。
- Flow 可以让首页统计和日历数据自动随打卡记录刷新。

### 2.3 版本路线

#### MVP 版本

- 单只或多只猫管理
- 内置常用行为：洗澡、驱虫、梳毛、剪指甲、喂药、称重、就医
- 用户自定义行为
- 一键打卡
- 首页展示上次打卡时间、距离今天多少天、近 30/90/180 天次数
- 月历视图展示每天打卡记录
- 本地数据库保存

#### V1.1

- 打卡频率设置
- 到期提醒
- 行为详情页
- 备注、照片
- 行为颜色和图标自定义

#### V2.0

- 云同步
- 家庭成员共享
- 多设备登录
- 数据导出
- 更丰富的统计图表

## 3. 功能模块

### 3.1 猫咪管理

用户可以创建和管理猫咪资料。

字段包括：

- 名字
- 头像
- 性别
- 生日或到家日期
- 品种
- 备注

操作包括：

- 新增猫咪
- 编辑猫咪信息
- 切换当前猫咪
- 删除猫咪，删除前二次确认

### 3.2 行为类型管理

行为类型是用户可以打卡的项目，例如洗澡、驱虫、梳毛。

内置行为：

- 洗澡
- 体内驱虫
- 体外驱虫
- 梳毛
- 剪指甲
- 喂药
- 称重
- 就医
- 清理耳朵
- 刷牙

自定义行为支持：

- 行为名称
- 图标
- 颜色
- 频率规则
- 是否在首页展示
- 是否开启提醒
- 备注说明

频率规则示例：

- 每 7 天一次
- 每 30 天一次
- 每 90 天一次
- 每周 3 次
- 每月 1 次
- 不固定，仅记录

### 3.3 手动打卡

用户在首页或行为详情页点击打卡按钮，即可生成一条记录。

打卡内容包括：

- 猫咪 ID
- 行为 ID
- 打卡时间
- 备注
- 图片，可后续版本加入
- 数值，可用于称重等行为

操作逻辑：

1. 用户点击某个行为卡片的打卡按钮。
2. 系统默认使用当前时间创建记录。
3. 若该行为需要补充信息，例如称重，则弹出输入面板。
4. 打卡成功后首页统计立即刷新。
5. 日历对应日期增加行为标记。

### 3.4 首页统计

首页是用户打开 App 后的主界面，重点展示当前猫咪的关键行为状态。

每个行为卡片显示：

- 行为名称
- 上次打卡日期
- 距离上次多少天
- 近 30 天次数
- 近 90 天次数
- 近 180 天次数
- 当前是否到期
- 一键打卡按钮

示例：

- 洗澡：距离上次 23 天，近 30 天 1 次，近 90 天 3 次
- 体外驱虫：距离上次 76 天，建议 90 天一次，还有 14 天
- 梳毛：本周 2/3 次

状态规则：

- 正常：未到频率周期
- 临近：距离到期小于等于 3 天
- 到期：达到设置频率
- 逾期：超过设置频率

### 3.5 日历视图

日历用于查看每天完成了哪些行为。

显示规则：

- 月视图展示整月日期
- 有打卡记录的日期显示彩色圆点或小图标
- 点击某一天展示当天记录列表
- 支持按行为筛选
- 支持切换月份

日历详情示例：

2026-05-19

- 洗澡 1 次
- 梳毛 1 次
- 称重 4.8kg

### 3.6 行为详情页

点击某个行为进入详情页。

展示内容：

- 行为名称、图标、颜色
- 上次打卡时间
- 当前频率设置
- 近 30/90/180 天次数
- 历史记录列表
- 编辑频率
- 添加补打卡

操作包括：

- 立即打卡
- 补录某一天的记录
- 删除错误记录
- 编辑备注

### 3.7 提醒通知

当行为达到用户设置的频率后，系统发送提醒。

实现方式：

- 使用 WorkManager 定期检查待提醒行为
- 使用 Notification 显示提醒
- 用户点击通知进入对应行为详情页

提醒规则：

- 可以单独开启或关闭
- 可设置提醒时间，例如每天 9:00
- 对于逾期行为，每天最多提醒一次

## 5. 页面结构

建议底部导航使用 4 个主入口：

- 首页
- 日历
- 行为
- 我的

### 5.1 首页

页面内容：

- 顶部猫咪切换区域
- 今日建议区域
- 常用行为卡片列表
- 最近记录

首页操作：

- 点击猫咪头像切换猫咪
- 点击行为卡片进入详情
- 点击打卡按钮完成记录
- 点击加号创建行为或补录记录

### 5.2 日历页

页面内容：

- 月份切换
- 月历网格
- 行为筛选
- 当天记录列表

日历操作：

- 左右切换月份
- 点击日期查看当天记录
- 点击记录进入详情
- 点击加号补录该日期记录

### 5.3 行为页

页面内容：

- 内置行为列表
- 自定义行为列表
- 是否首页展示开关
- 频率摘要

行为操作：

- 新增行为
- 编辑行为
- 启用/停用行为
- 设置频率
- 删除自定义行为

### 5.4 我的页

页面内容：

- 猫咪资料管理
- 提醒设置
- 数据备份，后续版本
- 关于 App

## 6. 操作逻辑

### 6.1 首次启动

1. 进入欢迎页。
2. 用户创建第一只猫。
3. 系统自动生成默认行为类型。
4. 进入首页。

### 6.2 创建自定义行为

1. 用户进入行为页。
2. 点击新增。
3. 输入名称。
4. 选择图标和颜色。
5. 选择频率规则。
6. 选择是否首页展示。
7. 保存后回到行为列表。

### 6.3 完成打卡

1. 用户在首页点击行为卡片右侧的打卡按钮。
2. 普通行为直接保存。
3. 需要数值的行为弹出输入框，例如称重。
4. 保存成功后显示轻提示。
5. 首页和日历自动刷新。

### 6.4 补录历史记录

1. 用户进入日历页。
2. 选择目标日期。
3. 点击新增记录。
4. 选择行为。
5. 填写备注或数值。
6. 保存。

### 6.5 删除记录

1. 用户进入行为详情或日期详情。
2. 长按或点击更多按钮。
3. 选择删除。
4. 二次确认。
5. 删除后重新计算统计。

## 7. 界面设计方案

### 7.1 设计风格

整体风格建议：温和、清爽、信息清晰。不要做成过度可爱的玩具感，重点是日常高频使用时好扫视、好点击、好记录。

视觉关键词：

- 干净
- 柔和
- 轻量
- 明确
- 有一点猫咪陪伴感

### 7.2 色彩建议

主色：

- 暖珊瑚色 #F47C6B

辅助色：

- 薄荷绿 #58BFA3
- 奶油白 #FFF8F2
- 深灰文字 #2E2E2E
- 次级文字 #7A7A7A
- 边框 #E8E1DA

状态色：

- 正常 #58BFA3
- 临近 #F5B84B
- 到期 #F47C6B
- 逾期 #D94A4A

### 7.3 首页布局

顶部：

- 当前猫咪头像
- 猫咪名字
- 切换猫咪按钮
- 新增记录按钮

中部：

- 今日建议列表，例如“体外驱虫还有 14 天”
- 逾期项目优先显示

主体：

- 行为统计卡片两列或单列展示
- 每张卡片包含图标、行为名、距离上次天数、近期次数和打卡按钮

底部：

- 首页、日历、行为、我的

### 7.4 行为卡片设计

卡片信息层级：

- 第一行：图标 + 行为名称 + 状态标签
- 第二行：距离上次多少天
- 第三行：近 30/90/180 天次数
- 右侧或底部：圆形打卡按钮

示例文案：

- 距上次 23 天
- 近 30 天 1 次
- 已到期
- 今天已打卡

### 7.5 日历设计

月历每个日期格子中展示：

- 日期数字
- 最多 3 个行为颜色点
- 超出显示 +N

点击日期后，底部弹出当天记录：

- 行为图标
- 行为名称
- 打卡时间
- 备注或数值

### 7.6 创建行为页面设计

表单字段：

- 行为名称输入框
- 图标选择
- 颜色选择
- 频率类型选择
- 频率值输入
- 首页展示开关
- 提醒开关
- 保存按钮

频率选择建议用分段控件：

- 不固定
- 每 N 天
- 每周 N 次
- 每月 N 次

## 9. 关键业务计算

### 9.1 距离上次打卡多少天

逻辑：

1. 查询某只猫某个行为最新一条记录。
2. 使用今天日期减去记录日期。
3. 没有记录时显示“暂无记录”。

### 9.2 近 30/90/180 天次数

逻辑：

1. 取当前时间。
2. 分别计算 30、90、180 天前的时间。
3. 查询时间范围内的记录数。
4. 首页卡片展示结果。

### 9.3 是否到期

对于每 N 天一次：

```text
距离上次天数 >= N，则到期
距离上次天数 >= N - 3，则临近
距离上次天数 > N，则逾期
```

对于每周 N 次：

```text
统计本周次数，少于 N 则未完成，达到 N 则完成
```

## 11. MVP 验收标准

MVP 完成时应满足：

- 用户可以创建至少一只猫。
- 用户可以使用默认行为进行打卡。
- 用户可以创建自定义行为。
- 用户可以设置行为频率。
- 用户可以在首页看到上次打卡距今天数。
- 用户可以看到近 30/90/180 天次数。
- 用户可以在日历中查看每天打卡记录。
- 用户可以补录和删除记录。
- 关闭 App 后数据不丢失。
