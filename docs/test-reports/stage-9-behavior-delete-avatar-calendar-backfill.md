# 阶段 9 测试报告：行为删除、猫咪头像与日历补录优化

## 范围

本阶段验证行为编辑页删除行为、删除关联记录策略、猫咪头像 URI、日历补录二级表单、构建和模拟器启动。

## 已实现内容

- 行为编辑页底部新增“删除行为”按钮。
- 删除行为前弹出二次确认，文案说明关联记录会一并删除。
- 删除行为时使用 Room `withTransaction` 同步删除行为和关联记录。
- 删除行为成功后返回行为清单，并清理无效行为详情状态。
- 猫咪编辑页新增“头像 URI”输入。
- 首页、猫咪列表和猫咪编辑页支持展示猫咪头像。
- 头像 URI 为空或加载失败时显示占位头像。
- 日历页补录入口改为“补录”按钮。
- 点击“补录”进入二级补录表单。
- 补录表单支持日期、行为、体重和备注。
- 补录保存后返回日历，取消后返回日历且不保存。

## 建议手工验证

1. 新增一个测试行为并创建关联记录。
2. 删除该行为并确认，验证首页、日历和行为详情不再显示该行为及关联记录。
3. 多猫场景下分别给猫 A、猫 B 创建同一行为记录，删除行为后验证两只猫都不再展示该行为记录。
4. 给当前猫填写有效头像 URI 并保存，验证首页、猫咪列表和编辑页展示一致。
5. 重启应用后确认头像 URI 保持或加载失败时显示占位头像。
6. 在日历二级补录表单提交普通行为和称重行为，验证保存后返回日历并刷新记录。
7. 在二级补录表单点击取消，验证不新增记录。
8. 切换猫咪后验证补录记录仍按当前猫隔离。

## 风险

- 行为删除级联记录已在代码层使用事务处理，但尚未真实执行验证。
- 头像当前是 URI 输入方案，不是系统图片选择器；无效 URI 只做占位兜底。
- 图片解码为轻量同步实现，后续建议接入专用图片加载库或系统 Photo Picker。
- 日历二级补录的保存、取消和多猫隔离还缺少完整手工验证。
- 当前没有自动化测试覆盖行为删除、头像持久化和日历二级补录。
