# 阶段 7 测试报告：记录展示与行为管理体验优化

## 范围

本阶段验证行为详情页和日历页记录列表的只读紧凑展示、三点菜单编辑/删除、行为管理清单优先入口、右上角新增按钮、中文图标选项和色块颜色选择。

## 已实现内容

- 行为详情页历史记录默认只读展示，不再直接显示编辑输入框。
- 日历当天记录列表默认只读展示，不再直接显示编辑输入框。
- 记录项使用更小边框、更小内边距和更紧凑行距。
- 每条记录右上角提供三点菜单。
- 三点菜单包含“编辑”和“删除”。
- 点击“编辑”后展开当前记录的编辑表单。
- 点击“删除”后继续使用二次确认。
- 行为管理页默认展示行为清单。
- 行为管理页右上角 `+` 按钮进入新增行为流程。
- 行为条目中的“编辑”按钮进入编辑行为流程。
- 行为新增和编辑不再展示图标 key 输入框，改为中文图标选项。
- 行为新增和编辑不再展示颜色代码，改为色块和中文颜色名。

## 建议手工验证

1. 启动 App，进入首页并打开任意行为详情。
2. 确认历史记录默认只读且布局紧凑。
3. 点击记录右上角三点菜单，确认包含“编辑”和“删除”。
4. 点击“编辑”，确认仅当前记录展开编辑表单。
5. 修改日期、时间、备注或体重后保存，确认详情页、首页统计和日历联动刷新。
6. 点击“删除”，确认出现二次确认，取消后记录保留，确认后记录删除。
7. 进入日历页，选择有记录日期，重复验证记录菜单、编辑和删除流程。
8. 进入行为管理页，确认默认只展示行为清单。
9. 点击右上角 `+`，确认进入新增行为流程。
10. 确认图标使用中文选项，颜色使用色块和中文颜色名。
11. 新增行为后确认返回清单，并且新行为展示正常。
12. 编辑已有行为，确认保存或取消后返回清单。

## 风险

- 记录编辑表单仍依赖手工输入 `yyyy-MM-dd` 和 `HH:mm`，格式错误提示需要重点手工验证。
- 三点菜单位于 `LazyColumn` 记录项中，需要验证快速点击和列表滚动时菜单状态不会串项。
- 行为图标当前为中文标签选择，尚未显示真实图形图标。
- 当前没有 Compose UI 自动化测试覆盖菜单、编辑展开和删除确认流程。
- `monkey` 启动后曾短暂回到 Launcher，显式启动复测正常，建议后续手工确认任务栈和首屏状态。
