# 阶段 8 测试报告：多猫管理闭环

## 范围

本阶段验证多猫资料管理、当前猫切换、猫咪资料编辑、删除二次确认、记录按当前猫隔离展示，以及 Debug 构建和模拟器启动。

## 已实现内容

- “我的”页新增猫咪资料管理入口。
- 猫咪资料列表展示全部猫咪，并标识当前猫咪。
- 支持新增多只猫咪，字段包含昵称、性别、生日、品种和备注。
- 支持切换当前猫咪。
- 支持编辑猫咪基础资料。
- 支持删除猫咪二次确认。
- 删除猫咪时同步删除该猫咪关联记录。
- 当前猫咪选择使用本地 `SharedPreferences` 持久化。
- 首页、日历、行为详情和记录列表按当前猫咪记录流刷新。
- 新增打卡和日历补录自动绑定当前猫咪。
- 无当前猫咪时，日历补录会提示先新增猫咪。

## UI 可达性验证

通过 `uiautomator dump` 和 adb 点击完成基础验证：

- 首页可见当前猫信息。
- 底部导航可见：首页、日历、行为、我的。
- 点击“我的”后进入猫咪资料页。
- 猫咪资料页可见标题“猫咪资料”、当前猫、右上角 `+`、当前猫卡片、“已选”“编辑”“删除”。
- 点击 `+` 后可进入“新增猫咪”表单。
- 新增表单可见昵称、男孩、女孩、未知、生日、品种、备注、保存和取消。
- 点击“删除”后可触发二次确认弹窗。
- 删除确认文案包含猫咪名称，并说明关联打卡记录会一并删除。

## 建议手工验证

1. 新增第二只和第三只猫，确认列表全部展示。
2. 在猫 A 下新增普通记录和称重记录。
3. 切换到猫 B，确认首页、日历和行为详情不显示猫 A 记录。
4. 在猫 B 下新增记录，再切回猫 A，确认猫 A 记录仍存在。
5. 编辑猫 A 资料，确认首页和“我的”页同步刷新。
6. 删除非当前猫，确认当前猫不变。
7. 删除当前猫，确认自动切换到剩余猫。
8. 删除最后一只猫，确认进入无猫空状态且应用不崩溃。
9. 重启应用，确认当前猫恢复逻辑符合预期。
